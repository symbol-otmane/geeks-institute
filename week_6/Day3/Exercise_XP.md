## Exercise 1: Communication & Role Clarity

### Team Roles & Responsibilities

| Role | Responsibilities |
|---|---|
| Game Designer | Define core gameplay loop, rules, balance, write design docs |
| Developer | Implement mechanics, physics, UI logic, fix bugs |
| Artist | Create visual assets, UI elements, placeholders |
| Producer | Task planning, deadline tracking, team coordination |

### Communication Channels

| Purpose | Tool |
|---|---|
| Daily communication | Discord |
| Task management | Trello |
| Documentation | Notion |
| Reviews & decisions | Weekly team meeting |

### Communication Plan
- **Daily:** Async check-in (done / next / blockers)
- **Weekly:** 20–30 min progress & planning meeting
- **Urgent issues:** Direct messages

## Exercise 2: Feedback & Iteration

### Feedback Framework
**Observation → Impact → Suggestion**

### Example Feedback
- **Observation:** Jump feels inconsistent on slopes  
- **Impact:** Platforming feels frustrating and unreliable  
- **Suggestion:** Improve ground detection or add landing buffer  

### Feedback Process
- Feedback shared during review sessions
- One action item selected per round
- Changes implemented and tested
- Results evaluated together

### Preventing Negative Feedback
- Focus on systems, not people  
- Use neutral, factual language  
- Avoid blame and absolute statements

## Exercise 3: Stress Management & Conflict Resolution

### Sources of Stress & Solutions

**1. Tight Deadlines**
- Split tasks into smaller milestones  
- Prioritize core mechanics over polish  

**2. Unclear Responsibilities**
- Assign one owner per task  
- Keep task board updated  

**3. Poor Communication**
- Daily short updates  
- Encourage early reporting of blockers  

---

### Conflict Scenario
Disagreement between designer and developer on mechanic implementation.

### Conflict Type
Task conflict

### Resolution Steps
1. Each side explains their view  
2. Identify shared goal (playable prototype)  
3. Compare solutions by time and scope  
4. Choose a testable compromise  
5. Document the decision  

### Leadership Behavior
- Stay neutral and calm  
- Encourage respectful discussion  
- Focus on solutions, not blame
